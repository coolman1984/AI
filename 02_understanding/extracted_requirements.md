# Extracted Requirements (Mission Analyst)

Real goal: an operational decision brain for a factory that reads files, computes exact
numbers, remembers relations and change over time, and presents audited, human-approved
management answers.

Current requirements live in `03_design/assistant_master_plan.md`; current Phase A work
lives in `03_design/phase_a_cards.md`. This file indexes them; it does not duplicate them.

- Explicit: extract/clean/store/analyze/search/report/dashboard; one department first,
  then factory.
- Implied: trust (no confident wrong number), in-house/offline, reviewability,
  restartability.
- Non-negotiable: independent audit + named human sign-off before anything reaches the CEO.
